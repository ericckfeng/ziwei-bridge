# 平滑參數動畫與宮位對比強化 — 設計文件

日期：2026-07-05
狀態：已核可（方案 A）

## 背景與目標

拓撲盤（`index.html`）用於觀察紫微斗數的數學關聯，並截圖分享至小紅書。現況兩個問題：

1. 只有「局數」有平滑旋轉動畫與自動運行；年干／年支／生月／生時改變時，`drawAux()` 清空重畫造成畫面「黑一下再重現」，無法目視追蹤星曜移動。
2. 十二宮底色為陰陽交替兩色、透明度僅 8%，宮與宮之間幾乎無法分辨，截圖對比不足。

目標：五個輸入參數皆可自動循環，且任何參數變動都以平滑動畫過渡；宮位在截圖中清楚可辨，整體風格（墨藍底／羊皮紙金）不變。

## 方案選擇

- **A. 擴充現有 rAF 補間引擎（採用）**：仿照 `goTo()`，以 `requestAnimationFrame` 每影格清空重畫插值狀態。與現有程式風格一致，easing 與 reduced-motion 邏輯共用。
- B. 持久節點＋CSS transition：否決——CSS transition 走直線而非圓弧，且與現有 rAF 機制並存兩套動畫。
- C. 全面狀態驅動渲染迴圈：否決——等於重寫渲染層，超出需求（YAGNI）。

## 設計

### 1. 循環控制 UI

- 控制列新增「循環對象」下拉選單：局數／年干／年支／生月／生時，預設「局數」。
- 「▶ 自動運行」與「停留秒數」滑桿對選定對象生效；`◀` `▶` 單步按鈕同樣作用於選定對象。
- 循環範圍：局數 0–11、年干 0–9（甲–癸）、年支 0–11、生月 1–12、生時 0–11；到底自動回頭（mod 循環）。

### 2. 動畫引擎

- `drawAux()` 改為接受「浮點位置表」（星名 → 渲染角度），與 `draw(tFloat)` 同構：每影格清空 `gAux` 重畫 13 顆吉煞星。
- 參數改變時的補間流程：
  1. 以 `auxPos()` 計算舊、新兩個狀態下每顆星的**最終渲染角度**（宮位 + 同宮散開偏移，單位為宮位浮點數）。
  2. 每顆星沿最短圓弧補間：角度差正規化到 [-6, 6] 宮。
  3. 時長 700ms，easing 為 `1-(1-k)^3`（與 `goTo()` 相同）。
- **四化徽章**：年干改變時，祿／權／科／忌四顆小圓徽章從舊星座標直線飛到新星座標。飛行中以獨立徽章（圓底＋單字）呈現於最上層；動畫結束後恢復現有的附著樣式（主星外圈光環＋「星名·祿」tspan）。四化目標可能是主星（主星環）或昌曲輔弼（吉煞環），起訖座標各自取該星當下的實際渲染座標。
- **重入處理**：動畫進行中再次變更任何參數時，取消進行中的 rAF，以當前插值狀態為新起點重新補間（不閃跳）。
- `prefers-reduced-motion` 為 reduce 時：跳過補間，直接落位（沿用現有邏輯）。
- 局數旋轉時吉煞星維持固定不動（保留現有行為——吉煞錨定出生四柱）。
- 側欄面板（本局結構／暗合列表）於動畫結束時更新一次，不逐影格更新。

### 3. 宮位色彩加強

- 扇形底色透明度 `.08` → `.16`。
- 陰色 `--yin: #4f7cd9` 微調為更深的藍（如 `#3d63c9` 一帶），拉開與陽橘 `#f2913d` 的視覺距離。
- 宮界線 `rgba(217,180,91,.15)` → `rgba(217,180,91,.4)`，線寬 `.6` → `1.2`。
- 地支文字不透明度 `.9` → `1`。
- 實際數值以截圖實測微調為準，原則：宮界清楚、底色分明、不搶連線與星曜的視覺焦點。

## 不做的事

- 不支援多參數同時循環（單一循環對象選單已足夠觀察）。
- 不改主星層 `draw()` 與連線邏輯。
- 不引入任何外部函式庫；維持單一 HTML 檔零依賴。

## 驗證

純前端視覺功能，以瀏覽器（本機 `python3 -m http.server` 或直接開檔）實測：

1. 五種循環對象逐一自動運行：無黑閃、星曜沿圓弧滑行、四化徽章飛行可追蹤。
2. 動畫中途改變另一參數：從當前位置平滑續接，不跳動。
3. 系統開啟「減少動態效果」：所有變化直接落位。
4. 截圖檢查：十二宮邊界與陰陽底色清楚可辨，連線仍為視覺主角。
5. 既有功能迴歸：局數旋轉、各勾選框開關、側欄資訊正確。
